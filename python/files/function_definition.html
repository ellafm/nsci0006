6. Function definition#

This chapter introduces the def keyword that we can use to define our own custom functions. We can create functions that have more than one input/output, and their construction can contain several lines of code.

After completing this chapter you should be able to create and use your own functions to solve problems.

6.1. Function basics#

The def keyword allows you to create functions that take a given input and return a calculated output. For example, the following code is used to define a function sq that returns the square of its input:

def sq(x):
  return x**2

Note

The syntax is important here. The first line ends with a colon, and any steps that are to be carried out by the function are indented.

Using the function

We can test out the behaviour of the function that we created using an integer as input. Here we label the returned variable as y :

y=sq(2)   
print("y=", y)

y= 4

Because this function returns a number when its input is a number, we can use it in place of a number in our code:

print(sq(3)+sq(10))

109

Exercise 6.1

Use the sq function to calculate:

  • \((999+123)^2\)

  • \(999^2+123^2\)

  • \(3^4\)

Solution

print(sq(999+123))
print(sq(999)+sq(123))
print(sq(sq(3)))

Passing a variable

We can pass a variable to the function as an argument, in much the same was as we have done when using built-in functions:

n=8
print(n, "squared is", sq(n))

8 squared is 64

Important!

Notice that the name of the variable that we passed to the function does not need to match the name that we chose when defining the function. The function uses whatever value we give to it in place of x. You can read more on this in the section below on local variables.

Local variables

The input variable names that we define when creating a function are local variables meaning that they only exist in the function and are not affected by any variable definitions in the rest of your script. They take their values from whatever inputs are given to the function.

For example, consider the code below in which the local x that appeared in our function definition takes its value from the input 3, and is unaffected by the variable definition that appeared in the line above it.

x=2                 #does not affect the function
print(sq(3))

9

Similarly, the script variable is unaffected by the function:

print(x)           #sq(3) does not affect variable x

2

Flexible functions

Functions are designed to generally work with multiple types of input. For example, the sq function that we created will also work with a numpy array:

x=np.arange(1,5,1)
print(sq(x))

[ 1  4  9 16]

However, you should take care as this can also lead to unexpected behaviour!

Exercise 6.2

Create and test a function that expects an array as input and returns the sum of the array as output. Your function should use a loop to calculate the sum of the array.

Solution

Here is an example of how it could be done:

def my_sum(x):
  T=0
  for k in x:
    T=T+k
  return T

#test
test_array = np.array([1,3,5,7,9])
print(my_sum(test_array))

6.2. Multiple inputs#

We can define functions that take more than one input. In the following example, we create a function that returns the value of the quadratic

(6.1)#\[\begin{equation} y=ax^2+bx+c \end{equation}\]

for a given variable array \(x\) and constants \(a,b,c\).

def quadratic(x,a,b,c):
  y = a*x**2 + b*x + c
  return y

Suppose that we then wish to use the quadratic function defined above to produce a plot of the curve

\[\begin{equation*} y=x^2-2x+3 \end{equation*}\]

Exercise 6.4

Explain why the following solution would be incorrect, and edit the code to make it correct:

a=1; b=-2; c=3

x = np.linspace(-4,4)   #choose a suitable range
y = quadratic(a,b,c,x)  #substitute the variable and parameters

plt.plot(x,y);

Solution

The proposed solution is incorrect, because the order of the inputs matters. In the function definition, the \(x\) values should appear first, followed by the coefficients in decreasing power order. Here is an example of how we can correct the code:

x = np.linspace(-4,4)   #choose a suitable range
y = quadratic(x,1,-2,3) #substitute the variable and parameters

plt.plot(x,y);

Further exercises

Exercise 6.5

Create and test a function npts(a,b,h) that returns the number of points in the interval \([a,b]\) assuming a common difference \(h\).

Solution

Here is an example solution. Note that if a step size is provided that does not subdivide the interval correctly then a solution is not possible!

The number of points needs to be converted to an integer if we want to use the output in the linspace function.

def npts(a,b,h):
  W = b-a           #width of interval
  s = W/h           #number of spaces
  n = int(s+1)      #number of points as an integer
  return n        

#test
start=1; stop=2; step=0.1
n = npts(start,stop,step)    #get result
x=np.linspace(start,stop,n)  #show that it is correct
print(x)

array([1. , 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 1.8, 1.9, 2. ])

Exercise 6.6

Modify your code from Exercise 3.3 to create a function my_logistic(t,C,t0,r)

Taking \(C=75\) and \(t_0=3\), use your function to produce plots of the logistic function for a range of \(r\) values (on a single figure).

Solution

Here is a tidy solution:

def my_logistic(t,C,t0,r):
  x= C/(1+np.exp(-r*(t-t0)))
  return x

#these values will be kept fixed
t0=3; C=75;
t=np.linspace(0,6,1000)

#changing r in a loop
for r in [0.5,1,3,10]:
  x=my_logistic(t,C,t0,r)
  plt.plot(t,x,label=f"$r= ${np.round(r,1)}")

plt.xlabel('t')
plt.ylabel('x')
plt.legend();

6.3. Multiple outputs#

We can create functions that return multiple outputs by arranging them in a list, as demonstrated by the following example:

def myf(x,y):
  return [x+y,x-y]

The behaviour then depends on the way that we use the function. If we call the function with a single output variable then the list object will be returned:

z = myf(1,2)
print(z)

[3, -1]

If we call the function with two output variables then the list values will be distributed to the variable names:

a,b=myf(1,2)
print(a)
print(b)

3
-1

Exercise 6.7

Create a function myfib(n) by adapting the solution code from Exercise 5.3. Your function should return two outputs:

  • the \(n\)th value in the Fibonacci sequence

  • the ratio given by dividing the \(n\)th value by the \((n-1)\)th value.

Solution

def myfib(n):
  x=np.empty(n)            #pre-allocate
  x[0]=1; x[1]=1           #first two values

  for i in range(1,n-1):
    x[i+1]=x[i]+x[i-1]     #iterative formula
  
  return [x[-1], x[-1]/x[-2]]

Notice that negative indexing is used to identify the elements at the end of the list. This helps to avoid confusion about which is the \(n\)th value that might occur due to Python indexing starting at zero.

If we want to get only one of the outputs we can do this by using an underscore _ to replace the output values that we do not want to keep. For example:

_,r=myfib(50)
print(r)

1.618033988749895

The ratio of the terms in the Fibonacci sequence converges to the golden ratio.
